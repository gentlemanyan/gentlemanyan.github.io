table布局必须知道的知识

Feb 20, 2017 | Hits

前言

  最近老大叫我修改一下网站的首页，将原来的highchart图表改为一种日历形式，可以每个日期显示该天一些相关信息。既然是一个日历形式，那我首先想到的就是表格布局的方法，一个td是一天，表头时周一到周日，这个思路貌似不错,想到了这些马上就动手开干了。

  下面是布局是遇到的问题：table布局有挺多坑的，因为我要兼容不同个浏览器，适应不同大小的屏幕。(其实如果用div+css可能更好一点,不用那么麻烦)

1、table宽高问题

  首先先说说两种表格布局方式

  table-layout：auto为自由布局，意味着表格的宽度和高度是自由的，随内容变化而变化。

  table-layout：fixed为固定布局，在固定表格布局中，水平布局仅取决于表格宽度、列宽度、表格边框宽度、单元格间距，而与单元格的内容无关。

  现在说说自由布局时，表格的默认行为是怎样的。

  table中设置的height其实是设置一个最小值，也就是当表格中的内容或者行高总值超过这个设置值时，会自动延长表格的height值，当表格中的内容或者行高没有达到这个值时，会自动扩大到这个值。

  table中设置的width值一般为表格宽度的最大值，如果td的内容太长且没有换行，那么table会优先计算内容长的td宽度，剩下的td如果设置了width则按照比例分配，没有设置width则按照内容计算。
（这个内部内容如果是图片的话是可以改变表格宽度的）

  table分配th、td的宽度的原则都是按照最宽的th或td先分配合适的宽度，然后再分剩下的th或td。

  table分配th、td的高度的原则是取最高的th或td的高度最为整个tr的高度。

2、tr的宽高问题

  tr设置width属性是无效的，因为table的width不能改变，所以tr的width无效。

  tr的height是由最大的那个td的高度决定的。

3、td的宽高问题

  td的width是最小宽度，td的内容没这么长，那么td宽度就是设置的宽度，如果内容过长，那么就是内容撑开的宽度。

  td的高度也是最小高度，就是td没有内容时候的最小的高度值。当内容过长，也会撑开td的高度，导致整行都变高。
